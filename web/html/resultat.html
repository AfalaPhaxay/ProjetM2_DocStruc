Présentation

Nous sommes 3 étudiantes en M2, Lili WU, Afala PHAXAY et Maeva LEPROUX.

Notre objectif pour ce projet était de prendre des données, de les structurer, et d'en obtenir des informations cohérentes et visuellement informatives.

Nous avons choisi de prendre comme données l'oeuvre Détective Conan. Nous avons récupéré les données, les avons re-structurées, et avons obtenu des statistiques intéressantes. Vous pourrez ainsi visualiser les différents thèmes abordés dans les chapitres de l'oeuvre, le nombre de morts dans les affaires criminelles et leur répartition par cause.

Cette page a été générée à partir de la transformation xsl d'un fichier xml avec xsltproc. Les fichiers xml et xsl se trouvent sur le dépôt git qui contient ce site.

Données

Nous avons récupéré nos données sur le site amateur d'informations sur l'oeuvre Détective Conan (https://www.detectiveconanworld.com/). Elles étaient donc au format HTML.

Nous les avons converties en XML grâce à un script comportant 4 fonctions:

  • extract_characters: extrait les personnages qui apparaissent pour la première fois dans l'oeuvre
  • extract_vd: extrait les victimes et personnes décédées
  • extract_cover: extrait la couverture du tome en question
  • extract: extrait tout le reste (chapitres, thèmes, résumés)

Voici ci-dessus un extrait du fichier XML.

Script pour l'obtention des données

Nous avons plusieurs fonctions pour extraire les données, car les données ayant été organisées de manière hasardeuse sur le site, c'était la solution la plus facile. Nous nous sommes servis du module BeautifulSoup, qui permet de manipuler les données HTML.

Nous avons ensuite essayé d'utiliser le module lxml pour organiser les données comme nous le désirions. Cependant, certaines informations étaient dupliquées dans la sortie xml du script, nous avons donc dû nous contenter de faire des print pour produire le fichier xml de sortie. Les deux scripts se trouvent dans le répertoire "script" du dépôt github.

Nos données sont en UTF-8, elles font 388Ko. Elles comportent les 90 tomes, c'est à dire 959 chapitres, avec leurs informations.

Vous pourrez retrouver, pour chaque tome:

  • sa couverture
  • ses noms de chapitres, avec leur thème
  • son résumé
  • ses résumés par thème (un peu plus précis)
  • les personnages qui y apparaissent pour la première fois et des informations les concernant
  • les victimes
  • les personnes décédées

Script utilisé pour les résultats

Détective Conan est une oeuvre qui, comme son nom l’indique, présente des affaires de meurtre ou d’autres affaires, parfois mineures, et le personnage principal Conan résout la majeure partie de ces affaires. Il y a donc de nombreuses personnes décédées au fil des pages. Nous avons pensé qu’il serait intéressant de pouvoir regarder quelle est la cause de mort qui revient le plus souvent dans l’oeuvre, et combien de personnes sont mortes de cette manière.

Nous avons donc écrit un script en python, aussi accessible sur le dépôt Github, pour obtenir les différentes statistiques sur les volumes de Détective Conan.

Nous avons ainsi pu obtenir des histogrammes et un pie chart en utilisant le module matplotlib ainsi que des données textuelles en parsant le fichier XML qui contient nos données avec lxml.

Aussi, nous avons observé que chaque chapitre (ou presque) a sa thématique propre: nous avons déjà utilisé ce site pour savoir quels chapitres faisaient partie de la trame principale (avec la Black Organization). Nous étions donc curieuses de savoir quelle pourcentage de chapitres y a t il avec ce thème, et pourquoi pas avec tous les thèmes!
